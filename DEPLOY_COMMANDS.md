# 🚀 RENDER DEPLOYMENT - STEP BY STEP WITH EXACT COMMANDS

## ⚡ 5-MINUTE DEPLOYMENT GUIDE

Copy and paste these exact commands to deploy to Render.

---

## Phase 1: Test Locally (2 minutes)

### Step 1A: Verify Build Works

```bash
npm run build
```

**Expected output:**

```
✓ your-frontend-assets compiled successfully
✓ typescript compiled successfully
```

### Step 1B: Test Production Server

```bash
npm start
```

**Expected output:**

```
Server is running on port 5000 in production mode
```

### Step 1C: Verify Deployment

Open browser:

```
http://localhost:5000
```

✅ You should see the dashboard login page

---

## Phase 2: Push to GitHub (2 minutes)

### Step 2A: Initialize Git (If Not Already Done)

```bash
git init
```

### Step 2B: Add All Files

```bash
git add .
```

### Step 2C: Create Initial Commit

```bash
git commit -m "Full-stack app ready for Render deployment"
```

### Step 2D: Create Main Branch

```bash
git branch -M main
```

### Step 2E: Add Remote Repository

Replace `YOUR-USERNAME` and `YOUR-REPO-NAME`:

```bash
git remote add origin https://github.com/YOUR-USERNAME/YOUR-REPO-NAME.git
```

### Step 2F: Push to GitHub

```bash
git push -u origin main
```

**You may be prompted for GitHub authentication:**

- Use GitHub username + personal access token (recommended)
- Or SSH key if configured

---

## Phase 3: Create Render Service (1 minute)

### Step 3A: Go to Render Dashboard

https://render.com/dashboard

### Step 3B: Create New Web Service

1. Click **"New +"**
2. Select **"Web Service"**

### Step 3C: Connect GitHub Repository

1. Click **"Connect a repository"**
2. Sign in with GitHub (if prompted)
3. Select your repository name
4. Click **"Connect"**

### Step 3D: Configure Service Settings

**Copy these exact values:**

| Setting           | Value                               |
| ----------------- | ----------------------------------- |
| **Name**          | `smart-leads-app`                   |
| **Environment**   | `Node`                              |
| **Region**        | `Oregon` (or your preferred region) |
| **Branch**        | `main`                              |
| **Build Command** | `npm run build`                     |
| **Start Command** | `npm start`                         |

### Step 3E: Add Environment Variables

1. Click **"Add Environment Variable"**
2. Add the following variables:

**Variable 1:**

```
Key: NODE_ENV
Value: production
```

**Variable 2:**

```
Key: JWT_SECRET_KEY
Value: <generate-a-strong-random-secret>
```

**To generate a secure JWT secret, use any of these:**

```bash
# Option 1: OpenSSL (macOS/Linux)
openssl rand -base64 32

# Option 2: Node.js
node -e "console.log(require('crypto').randomBytes(32).toString('hex'))"

# Option 3: Online (use: https://randomkeygen.com/ - Fort Knox password)
```

### Step 3F: Review Settings

Verify all settings match above.

### Step 3G: Create Web Service

Click **"Create Web Service"**

**Render will now:**

1. Clone your repository
2. Build the frontend (Vite)
3. Build the backend (TypeScript)
4. Start the Express server
5. Deploy the application

---

## 📊 COMMON ISSUES & FIXES

### Issue: Build Fails with TypeScript Error

```
1. Check error message in Render Logs
2. Fix TypeScript error locally
3. Commit and push: git push origin main
```

### Issue: Port Already in Use (Local Testing)

```bash
# Kill process on port 5000
# macOS/Linux:
lsof -ti:5000 | xargs kill -9

# Windows (PowerShell as Admin):
Get-Process -Id (Get-NetTCPConnection -LocalPort 5000).OwningProcess | Stop-Process -Force
```

### Issue: Environment Variables Not Working

```
1. Verify in Render Dashboard → Your Service → Environment
2. Ensure JWT_SECRET_KEY is set
3. Restart service: Render Dashboard → Deployments → Manual Deploy
```

### Issue: Blank Page on Live Site

```
1. Open DevTools (F12)
2. Check Console for JavaScript errors
3. Check Render Logs for Express errors
4. Verify vite.config.ts has correct outDir
5. Local test: npm run build && npm start
```

## 📞 COMMAND REFERENCE

```bash
# Development
npm run dev                 # Start frontend + backend
npm run server             # Start backend only
npm run client             # Start frontend only

# Production (Render)
npm run build              # Build for production
npm start                  # Start production server

# Git
git add .                  # Stage changes
git commit -m "message"    # Commit changes
git push origin main       # Push to GitHub

# Local Testing
npm install                # Install dependencies
npm run install:all        # Install all dependencies
```
